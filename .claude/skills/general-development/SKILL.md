# General Development Skill

**Purpose:** Universal software development patterns and best practices applicable across all programming languages and frameworks.

## When to Use This Skill

- General programming tasks and architecture decisions
- Code organization and structure questions
- Development workflow and process improvements
- Cross-platform or multi-language development
- Software design patterns and principles
- Best practices for maintainable code

## Key Development Areas Covered

### Code Architecture
- **Design Patterns**: SOLID principles, GoF patterns, architectural patterns
- **Code Organization**: Module structure, separation of concerns, dependency management
- **Refactoring**: Code improvement techniques, legacy code modernization
- **Testing**: Unit testing, integration testing, test-driven development

### Development Workflow
- **Version Control**: Git workflows, branching strategies, merge practices
- **Code Review**: Peer review processes, quality standards, feedback guidelines
- **Documentation**: Code documentation, API docs, README files
- **Development Environment**: Setup, tooling, configuration management

### Performance & Quality
- **Performance Optimization**: Profiling, bottlenecks, optimization strategies
- **Code Quality**: Linting, static analysis, code metrics
- **Security**: Secure coding practices, vulnerability prevention
- **Debugging**: Systematic debugging, issue resolution techniques

### Project Management
- **Agile Practices**: Sprint planning, retrospectives, continuous improvement
- **Requirements Analysis**: Feature specifications, user stories, acceptance criteria
- **Technical Debt**: Identification, prioritization, management strategies

## Universal Principles

### Code Quality
- **Readability**: Clear, self-documenting code with meaningful names
- **Maintainability**: Easy to modify, extend, and understand
- **Testability**: Code designed for easy testing and validation
- **Scalability**: Architecture that handles growth and change

### Development Practices
- **Incremental Development**: Small, frequent, testable changes
- **Continuous Integration**: Automated testing and integration
- **Code Reviews**: Peer feedback and knowledge sharing
- **Documentation**: Living documentation alongside code

### Problem-Solving Approach
- **Systematic Analysis**: Understanding problems before implementing solutions
- **Iterative Improvement**: Refining solutions based on feedback
- **Knowledge Sharing**: Documenting decisions and lessons learned
- **Collaboration**: Working effectively with team members

## Technology-Agnostic Patterns

### Modular Design
- **Single Responsibility**: Each module has one clear purpose
- **Loose Coupling**: Minimal dependencies between components
- **High Cohesion**: Related functionality grouped together
- **Interface Segregation**: Small, focused interfaces

### Error Handling
- **Graceful Degradation**: Systems handle failures elegantly
- **Explicit Error Handling**: Clear error states and recovery paths
- **Logging**: Comprehensive logging for debugging and monitoring
- **User Experience**: Meaningful error messages and recovery options

### Data Management
- **Data Integrity**: Validation and constraints to ensure data quality
- **Efficient Access**: Optimized data retrieval and storage
- **Security**: Proper data protection and access controls
- **Backup & Recovery**: Strategies for data preservation
